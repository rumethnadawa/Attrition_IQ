Data Pre-processing Steps

  • Feature Removal: Dropped constant/unhelpful columns such as EmployeeCount, EmployeeNumber, Over18, and StandardHours.
  • Target Encoding: Encoded the target variable 'Attrition' (Yes=1, No=0).
  • Label Encoding: Applied LabelEncoding to categorical columns like BusinessTravel, Department, EducationField, Gender, JobRole, and MaritalStatus.
  • Data Splitting: Splitted data into 80% training and 20% testing sets using stratified sampling to maintain class proportions.
  • Class Imbalance Handling: Applied Synthetic Minority Over-sampling Technique (SMOTE) strictly on the training set to create a balanced dataset (50/50).
  • Feature Scaling: Applied Standard Scaling on features for distance-based models (Logistic Regression, SVM). Tree-based models used unscaled data.

Reasons for Model Selection

Random Forest was selected as the final model because it exhibited the highest and most stable Cross-Validation AUC (~0.96) and solid precision/recall balance. Tree-based models are also robust to outliers and require less strict feature scaling.

Hyperparameter tuning via RandomizedSearchCV was applied to the Random Forest model to optimize depth, estimators, and splits.
